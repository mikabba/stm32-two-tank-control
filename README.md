# Water Tank Control with STM32



Embedded control project for regulating the water level in a two-tank hydraulic system using an STM32 microcontroller.



The firmware runs on an **STM32 NUCLEO-F411RE** board and acquires water-level measurements from two **VL53L0X Time-of-Flight sensors**. The measured levels are filtered and used by digital controllers implemented on the microcontroller. The control output drives a 12 V DC pump through an **HW-095 / L298N motor driver** using a PWM signal.



---



## Project Overview



The goal of this project is to design and implement a complete embedded control system for a water tank plant.



The project includes:



- construction of the physical two-tank hydraulic system;

- water-level measurement using VL53L0X laser-ranging sensors;

- mathematical modelling and linearization of the plant;

- design and implementation of PID and LQI controllers;

- PWM-based actuation of a DC water pump;

- experimental validation on the real plant.



The controlled variable is the water level in the second tank. The STM32 periodically reads the sensors, computes the control action and updates the pump duty cycle.

## Firmware Modules



### `main.c`



Main application file. It initializes peripherals, sensors, tanks, filters, pump and controllers.  

The main loop reads sensor data, maps the measured distances into water levels and updates the filtered tank levels.



### `sensors.c` / `sensors.h`



Sensor management module for the two VL53L0X Time-of-Flight sensors.



Main responsibilities:



- initialization of multiple VL53L0X sensors on the same I2C bus;

- XSHUT-based sensor boot sequence;

- assignment of different I2C addresses;

- acquisition of distance measurements;

- conversion from measured distance to tank water level.



### `Motor.c` / `Motor.h`



Pump actuation module.



Main responsibilities:



- initialization of the pump structure;

- conversion from controller output to PWM duty cycle;

- duty-cycle saturation;

- update of the timer compare register used for PWM generation.



### `control.c` / `control.h`



Control module containing the implemented digital controllers.



Implemented controllers:



- PID controller;

- LQI controller.



The controller output is the duty cycle applied to the pump.



### `MovingAverageFilter.c` / `MovingAverageFilter.h`



Moving average filter used to reduce noise and fluctuations in the measured water level signals.



---



## Control Architecture



The system is controlled digitally by the STM32.



At each sampling instant, the firmware performs the following operations:



1. read the distance from both VL53L0X sensors;

2. convert the measured distances into water levels;

3. filter the measured levels using a moving average filter;

4. compute the control action using the selected controller;

5. saturate the control output between 0% and 100%;

6. update the PWM duty cycle applied to the pump driver.



The controlled output is the water level of **Tank 2**.

## Peripherals Configuration



| Peripheral | Purpose |

|---|---|

| I2C | Communication with the VL53L0X sensors |

| TIM2 | PWM generation for pump actuation |

| TIM1 | Periodic timing for the control action |

| GPIO | Pump direction control and VL53L0X XSHUT pins |

| USART | Debug and serial communication |



---



## VL53L0X Sensor Management



The two VL53L0X sensors are connected to the same I2C bus.



Since both sensors have the same default I2C address, they are initialized one at a time using their XSHUT pins. After boot, each sensor is assigned a different I2C address.



| Sensor | I2C Address |

|---|---|

| VL53L0X 1 | `0x54` |

| VL53L0X 2 | `0x56` |



The sensors are configured in high-accuracy mode to improve the precision of the water-level measurement.



---



## Pump Actuation



The pump is driven through an **HW-095 / L298N** driver.



The STM32 controls the pump using:



- one PWM signal for speed control;

- two GPIO output pins for driver input control.



The PWM signal is generated using **TIM2**.



PWM configuration:



| Parameter | Value |

|---|---|

| PWM frequency | approximately 10 kHz |

| Timer | TIM2 |

| Duty cycle range | 0% to 100% |



The controller output is converted into a duty cycle and written to the timer compare register.



---



## Controllers



Two control strategies are implemented in the firmware.



### PID Controller



The PID controller computes the pump duty cycle from the error between the reference level and the measured level of Tank 2.



The implemented PID uses:



| Parameter | Value |

|---|---|

| `Kp` | 1.8410 |

| `Ki` | 0.0118 |

| `Kd` | 60.8833 |



The controller output is saturated between 0% and 100%.



### LQI Controller



The Linear Quadratic Integral controller uses state feedback with integral action to reduce steady-state error and improve tracking performance.



The implemented LQI gains are:



| Parameter | Value |

|---|---|

| `K1` | 15.7345 |

| `K2` | 23.1724 |

| `K3` | -0.3162 |



The controller selection can be changed in the timer callback function inside `main.c`.



---



## Sampling Time



The control action is computed periodically with a sampling time of:



```text

Ts = 1 s

```



This value was chosen considering the slow dynamics of the hydraulic system and the response time of the real plant.



---



## Plant Model



The system consists of two coupled tanks.



The state variables are:



| Variable | Description |

|---|---|

| `x1` | water level in Tank 1 |

| `x2` | water level in Tank 2 |



The plant is nonlinear because the outlet flow depends on the square root of the water level, according to Torricelli's law.



A linearized model was obtained around an equilibrium point and used for controller design and simulation.



---



## Experimental Results



The controllers were tested both in simulation and on the real plant.



The LQI controller designed on the mathematical model showed good tracking performance on the physical system.



A system identification procedure was also performed using an APRBS input signal. The identified model was then used to tune a PID controller that achieved better agreement between simulation and real plant response.



The identified PID controller showed a settling time close to the simulated response and better disturbance rejection compared to the controller designed only from the mathematical model.

## Build Instructions



### Requirements



To build and flash the project, the following tools are required:



- STM32CubeIDE

- STM32 NUCLEO-F411RE board

- ST-LINK debugger

- USB cable

- ARM GCC toolchain, included with STM32CubeIDE



---



### Open the Project in STM32CubeIDE



1. Clone the repository:



```bash

git clone https://github.com/<username>/<repository-name>.git

```



2. Open STM32CubeIDE.



3. Select:



```text

File > Import > Existing Projects into Workspace

```



4. Select the cloned repository folder.



5. Import the project.



6. Build the project using the `Debug` configuration.



---



## Flashing the Firmware



Connect the STM32 NUCLEO-F411RE board through USB.



From STM32CubeIDE, use:



```text

Run > Debug

```



or:



```text

Run > Run

```



The firmware can also be flashed using STM32CubeProgrammer if a `.elf`, `.hex` or `.bin` file is generated locally.
